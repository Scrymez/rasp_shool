# Аманат Расписание

## Автор

- Автор: **Латипов Саид Ахмедович**
- Разработчик: **Латипов Саид Ахмедович**
- Название ПО: **Аманат Расписание**
- Назначение: автоматическое создание и сопровождение школьного расписания.

`Аманат Расписание` — локальное desktop-приложение для автоматического создания школьного расписания. Название связано с понятием **аманат**: доверенная ответственность, которую нужно выполнить честно и аккуратно.

## Возможности

- создание классов 1-11 с уровнями НОО, ООО, СОО;
- поддержка 1 и 2 смены;
- импорт предметов;
- импорт учителей из Excel;
- ручное добавление учителя, ФИО и нескольких предметов;
- привязка предметов к параллелям;
- настройка часов в неделю по каждой параллели;
- привязка учителей к классам и предметам;
- настройка учебных дней, выходных, уроков и перемен;
- автоматическая генерация расписания с учетом сложности предметов;
- экспорт расписания в Excel и PDF;
- шаблоны Excel для учителей и расписания;
- отчеты по учителям, классам, предметам и классным руководителям;
- локальная SQLite-база;
- Windows installer с ярлыком на рабочем столе;
- проверка обновлений через GitHub Releases.

## Установка ПО на Windows

### Системные требования

- Windows 10 или Windows 11, 64-bit.
- 4 ГБ ОЗУ минимум, лучше 8 ГБ.
- 500 МБ свободного места для приложения.
- Отдельное место для базы, backup и экспортов расписания.
- Права обычного пользователя достаточно, если установка идет в пользовательскую папку.
- Интернет нужен только для скачивания установщика и проверки обновлений.

### Что устанавливается автоматически

Desktop-версия не требует ручной установки Node.js, Chrome, SQLite или сервера. Все нужные компоненты входят в installer:

- Electron desktop-оболочка;
- встроенный Node.js runtime;
- встроенный Chromium;
- локальный Express API;
- SQLite-база через встроенный Node runtime;
- React frontend;
- система обновлений.

### Установка для обычного пользователя

1. Скачать установщик из GitHub Releases:

```text
amanat-raspisanie-setup-<version>.exe
```

2. Запустить `.exe`.
3. Выбрать папку установки или оставить путь по умолчанию.
4. Дождаться завершения установки.
5. Запустить приложение через ярлык:

```text
Аманат Расписание
```

6. Войти как администратор.

Пароль по умолчанию:

```text
admin
```

После первого входа нужно сразу сменить пароль во вкладке `Система`.

### Первый запуск

При первом запуске приложение автоматически создает рабочие папки:

```text
%APPDATA%/Аманат Расписание/data
%APPDATA%/Аманат Расписание/exports
%APPDATA%/Аманат Расписание/logs
%APPDATA%/Аманат Расписание/updates
```

Назначение папок:

- `data` — локальная SQLite-база;
- `exports` — файлы расписания и отчеты;
- `logs` — служебные журналы;
- `updates` — временные файлы обновлений.

База desktop-версии:

```text
%APPDATA%/Аманат Расписание/data/scheduler.db
```

Статус встроенных компонентов виден во вкладке `Система` -> `Компоненты Windows`.

### Порядок настройки после установки

1. Войти с паролем администратора.
2. Сменить пароль.
3. Создать классы 1-11 и указать смену.
4. Добавить или импортировать предметы.
5. Настроить часы предметов по параллелям.
6. Импортировать или вручную добавить учителей.
7. Привязать учителей к классам и предметам.
8. Назначить классных руководителей.
9. Настроить учебные дни, смены, уроки и перемены.
10. Создать расписание.
11. Проверить отчеты.
12. Сохранить backup.

### Обновление установленной версии

Когда репозиторий с релизами будет публичным, приложение сможет проверять обновления через GitHub Releases.

В приложении есть кнопка `Обновления` справа сверху:

- `Проверить` — ручная проверка новой версии;
- `Скачать` — скачивание найденной версии;
- `Установить` — перезапуск и установка скачанного обновления.

Установленная версия автоматически проверяет обновления через 5 секунд после запуска.

Для обновлений в публичном репозитории должны быть assets:

- `amanat-raspisanie-setup-<version>.exe`;
- `amanat-raspisanie-setup-<version>.exe.blockmap`;
- `latest.yml`.

### Удаление приложения

Удалить приложение можно через стандартный раздел Windows:

```text
Параметры -> Приложения -> Установленные приложения -> Аманат Расписание -> Удалить
```

Важно: удаление приложения может не удалить базу из `%APPDATA%`. Перед удалением сделайте backup во вкладке `Система`.

### Перенос на другой компьютер

1. На старом компьютере открыть вкладку `Система`.
2. Скачать backup.
3. Установить `Аманат Расписание` на новом компьютере.
4. Открыть вкладку `Система`.
5. Восстановить backup.
6. Проверить классы, предметы, учителей и последнее расписание.

## Обновления через GitHub

Репозиторий: `https://github.com/Scrymez/rasp_shool.git`.

Репозиторий должен быть публичным для нормальной работы автообновлений через GitHub Releases без GitHub token на компьютере пользователя.

Сборка релиза через тег:

```powershell
npm version patch
git push
git push --tags
```

После push тега `v*` GitHub Actions собирает Windows installer и прикрепляет к GitHub Release:

- `amanat-raspisanie-setup-<version>.exe`;
- `amanat-raspisanie-setup-<version>.exe.blockmap`;
- `latest.yml`.

### Требования для автообновлений

Для автообновления нужны:

- публичный GitHub repo;
- GitHub Release с тегом `v*`;
- `latest.yml` в assets релиза;
- installer `.exe` в assets релиза;
- `.exe.blockmap` в assets релиза;
- версия в `package.json` выше установленной версии;
- установленное приложение собрано с GitHub publish config.

### Проверка автообновления

После выпуска новой версии пользователь открывает приложение и нажимает:

```text
Обновления -> Проверить -> Скачать -> Установить
```

Пользователю не нужен GitHub token.

## Данные

- база разработки: `data/scheduler.db`;

- документы Obsidian: `docs/`;

## Импорт Excel

- сотрудники: 1 столбец `ФИО`, дальше предметы или 2 столбец `предмет`;
- предметы: `Предмет`, `Уровни`, `Классы`, `Сложность`, `Часы`;
- готовое расписание: `Класс`, `День`, `Урок`, `Предмет`, `Учитель`.

## Документы для пользователей перед работой

Перед использованием приложения в школе пользователи должны подписать или принять:

- **Пользовательское соглашение** — правила доступа, запрет передачи пароля, ответственность за корректность введенных данных.
- **Политика обработки персональных данных** — ФИО учителей, классы, нагрузка, расписание, хранение локальной базы.
- **Согласие на обработку персональных данных сотрудников** — если в приложение вносят ФИО учителей и нагрузку.
- **Регламент информационной безопасности** — кто имеет доступ к компьютеру, backup, паролям, экспортам Excel/PDF.
- **Инструкция администратора** — как импортировать данные, создавать расписание, делать backup, восстанавливать базу.
- **Регламент резервного копирования** — где хранить backup, как часто делать копии, кто отвечает за восстановление.
- **Акт приема-передачи/внедрения ПО** — кто получил приложение, версия, дата установки, ответственный администратор.
- **Лицензионные условия/условия использования** — право использования приложения, автор: Латипов Саид Ахмедович.
- **Журнал изменений и версий** — что изменилось в релизах, когда обновляли приложение.

Дополнительно лучше разработать:

- краткую инструкцию для директора;
- инструкцию для завуча/администратора расписания;
- шаблон приказа о назначении ответственного за расписание и данные;
- форму согласия сотрудников на обработку персональных данных;
- политику хранения экспортов Excel/PDF.
