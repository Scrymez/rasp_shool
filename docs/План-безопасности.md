# План безопасности приложения

## Цель

Защитить локальное desktop-приложение расписания от удаленного доступа к базе, несанкционированного восстановления backup, кражи школьных данных и случайного открытия API в сеть.

## Уровень 1. Критичная защита

- Закрыть все `/api/*` методы токеном администратора, кроме `/api/login` и `/api/health`.
- Хранить пароль администратора только в виде `scrypt`-хеша с солью.
- Автоматически мигрировать старый открытый пароль из базы в хеш.
- Запретить backup, restore, отчеты, экспорт и импорт без токена администратора.
- Защитить новые импорты классов, классных руководителей, блокировок и шаблоны тем же admin-токеном.
- Валидировать Excel-импорт классов, руководителей и расписания перед записью в базу.
- Ограничить CORS только локальными адресами `127.0.0.1` и `localhost`.
- По умолчанию запускать сервер только на `127.0.0.1`.
- Не разрешать `HOST=0.0.0.0` без явного флага `SCHEDULER_ALLOW_NETWORK=1`.

Статус: выполняется в версии 0.1.9.

## Уровень 2. Защита от подбора и ошибок администратора

- Добавить ограничение попыток входа: задержка или блокировка после нескольких неверных паролей.
- Добавить обязательную смену стандартного пароля `admin` при первом входе.
- Добавить проверку сложности нового пароля: длина, цифры, буквы, спецсимволы.
- Добавить отдельный журнал безопасности: вход, выход, смена пароля, backup, restore, импорт.
- Показывать администратору предупреждение, если используется стандартный пароль.

## Уровень 3. Защита данных

- Сделать экспорт backup с опциональным паролем и шифрованием.
- Перед restore показывать экран подтверждения с количеством таблиц и датой backup.
- Добавить автоматический локальный backup перед restore.
- Исключить служебные секреты из обычного backup или переносить их отдельно.
- Добавить контроль размера импортируемых файлов и понятные ошибки по поврежденным файлам.
- Добавить предварительный просмотр импортируемых классов/руководителей до применения изменений.

## Уровень 4. Защита desktop-сборки

- Подписать Windows-установщик сертификатом Code Signing.
- Включить проверку подписи обновлений после появления сертификата.
- Хранить токен только в `sessionStorage`, не в `localStorage`.
- Оставить Electron с `contextIsolation: true`, `nodeIntegration: false`, `sandbox: true`.
- Запретить открытие внешних ссылок внутри окна приложения.

## Уровень 5. Процесс безопасности

- Добавить регулярный `npm audit`.
- Добавить smoke-тест, который проверяет отказ API без токена.
- Перед релизом проверять: build, smoke, audit, выпуск установщика.
- Вести список известных рисков в этом документе.

## Что остается после уровня 1

- Rate limit на вход.
- Принудительная смена стандартного пароля отдельным экраном.
- Шифрованный backup.
- Подпись Windows-установщика.
- Отдельный отчет безопасности для администратора.
- Предпросмотр Excel-импорта перед записью.
