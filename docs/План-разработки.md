# План разработки автоматического расписания

## Цель

Локальное веб-приложение для администратора школы: импорт данных, настройка классов и недель, автоматическое создание расписания, экспорт в Excel.

## Архитектура

- Frontend: React + Vite.
- Backend: Node.js + Express.
- База: SQLite файл `data/scheduler.db`.
- Excel: импорт и экспорт `.xlsx` через минимальный zip/xml слой на `fflate`.

## Правила генерации v1

- Сложные предметы ставятся преимущественно на 1-4 уроки.
- Легкие предметы ставятся ближе к середине или концу дня.
- Один предмет не повторяется в один день, если есть свободные дни.
- Учитель не ставится в два класса на один и тот же урок.
- Кабинет не ставится двум классам на один и тот же урок.
- Ограничения учителя по дню или уроку блокируют слот.
- СанПиН-профиль ограничивает максимум уроков и суммарную сложность за день.
- Нагрузка распределяется по дням с меньшим числом уроков.
- Класс учится в одной из смен: 1 смена `morning`, 2 смена `afternoon`.
- Время урока зависит от смены класса и стартового времени смены.
- Учитель и кабинет могут быть заняты в один номер урока в разных сменах, если реальные смены не пересекаются.
- Часы предмета задаются отдельно для каждой параллели 1-11.
- Если предмет не привязан к параллели, он не создается в связках этого класса.
- При создании связок `класс-предмет` количество часов берется из настройки параллели.
- Классный руководитель задается отдельно от предметных связок и попадает в backup/restore и отчеты.
- Сложность предмета задается по внутренней шкале 1-5 на основе ФГОС/ФОП-предметов, недельной нагрузки и типа учебной деятельности.
- Генератор делает несколько проходов с разными стратегиями и выбирает расписание с лучшим score.
- Score штрафует незапланированные уроки, окна класса, сложные предметы поздно, повторы предмета в день и дисбаланс сложности.
- Конфликты учителя и кабинета считаются по реальному времени урока, а не только по номеру урока.

## Правила сложности предметов

- `1` — разгрузочные двигательные и творческие предметы: физическая культура, музыка, ИЗО.
- `2` — практические предметы с умеренной теорией: технология, ОРКСЭ, ОДНКНР.
- `3` — средняя нагрузка: окружающий мир, ОБЗР, индивидуальный проект, второй иностранный язык.
- `4` — высокая текстовая или понятийная нагрузка: литература, история, обществознание, география, биология, информатика, иностранный язык.
- `5` — максимальная абстрактная, языковая или расчетная нагрузка: русский язык, математика, алгебра, геометрия, физика, химия.
- Если школа считает предмет сложнее или легче из-за профиля класса, сложности можно менять вручную во вкладке `Предметы`.

## Отчеты

- `Учителя`: суммарные часы, предметы, классы.
- `Учитель-классы`: каждая связка учитель-класс-предмет-часы-кабинет.
- `Классы`: уровень, смена, часы, количество предметов, список учителей.
- `Класс-предметы`: предметная нагрузка каждого класса.
- `Классные руководители`: класс и назначенный руководитель.
- `Кабинеты`: тип, вместимость, количество назначений, классы.
- `Проблемы`: предметы без учителя, без кабинета, незапланированные уроки и окна учителей.

## Обновления приложения

- Источник релизов: GitHub Releases `Scrymez/rasp_shool`.
- Publish-команда CI: `npm run release:win`.
- Trigger CI: push tag `v*` или ручной `workflow_dispatch`.
- Runtime: Electron `autoUpdater`.
- UI: кнопка `Обновления` в верхней панели.
- Установленная версия делает авто-проверку через 5 секунд после запуска.
- Для приватного repo безопасный production-вариант требует публичный release repo или update-server/proxy, чтобы не вшивать GitHub token в `.exe`.

## Безопасность

- План безопасности: `docs/План-безопасности.md`.
- Выполнить уровень 1: токен администратора для API, хеш пароля, защита backup/restore/export/import, локальный CORS, локальная привязка сервера.
- После уровня 1 осталось: rate limit входа, принудительная смена стандартного пароля отдельным экраном, шифрованный backup, подпись Windows-установщика, отдельный отчет безопасности.

## Дизайн

Направление: магическая школьная канцелярия. Администратор работает как с гримуаром расписания: темная база, пергаментные панели, золотые линии, мятные акценты, сигил входа, расписание как созвездная сетка.
